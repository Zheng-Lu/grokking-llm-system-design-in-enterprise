# Security Policy

## Reporting a vulnerability

If you discover a security issue in this repository's tooling, workflows, or published site, please do not open a public issue first.

Instead, report it through one of these private channels:

- GitHub Security Advisories, if enabled on the repository
- email: `zheng.lu@gmail.com`

Please include:

- a short description of the issue
- affected files or workflows
- reproduction steps
- impact assessment
- any suggested mitigation

## What is in scope

This is a static documentation repository, so the main security surface is limited to:

- GitHub Actions workflows
- helper scripts
- dependency updates
- site configuration

Content quality issues, citation disputes, and editorial disagreements should go through normal issue or pull request channels instead.

## Response expectations

Maintainers should aim to:

- acknowledge receipt within 5 business days
- assess severity and scope
- prepare a fix or mitigation
- publish a coordinated disclosure note when appropriate

## Secrets and sensitive content

Do not commit:

- credentials or tokens
- internal-only case study material
- confidential diagrams or screenshots
- proprietary source material without clear permission
