# Platform

This subchapter covers the shared control-plane and operational services around model access. The emphasis is routing, policy, observability, and platform consistency across applications.

## In this subchapter

- [Platform Foundations](platform-foundations.md) defines the shared infra layer and the main design tensions it introduces.
- [Design an LLM Gateway / Proxy](llm-gateway-proxy.md) focuses on routing, policy enforcement, and platform control at the model boundary.
- [Design an Evaluation & Observability System for LLM Applications](evaluation-observability-system.md) covers tracing, datasets, release gates, and regression detection.

## Reading lens

Use this section when the architecture is about control, visibility, and consistency across multiple inference clients.
