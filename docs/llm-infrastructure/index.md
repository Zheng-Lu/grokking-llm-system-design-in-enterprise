# LLM Infrastructure

This section covers the serving and control layer underneath most LLM applications. It is where teams turn model access into a shared infrastructure layer with latency targets, quotas, routing policy, traceability, and release discipline.

## In this chapter

- [Inference](inference/index.md) groups the batch and online serving paths.
- [Platform](platform/index.md) groups the shared control-plane and operational services.

## Section lens

If the model is the engine, this section is the transmission, brakes, and dashboard.
